Ecosystem Navigation: 3 Approaches

How child projects appear on a parent's detail page + how relationships are managed

A

Inline Card Strip + Edit Modal

Child grid: Horizontal strip of minimal cards above tabs. Scrollable if many children.

Management: "Manage Sub-Projects" button opens a modal with search-to-add and remove buttons.

Pros: Clean separation. Management is tucked away. Minimal changes to existing layout.

Cons: Horizontal scroll can hide children. Modal adds a layer of indirection for management.

B

Card Grid + Inline Actions

Child grid: Wrapped grid (not scrolling) of minimal cards above tabs. 3-4 columns like the main projects view.

Management: "+" button to add children (search popover). "×" on each card to unlink. On any project card elsewhere: right-click → "Set Parent..."

Pros: All children visible at once. Direct manipulation — no modal needed. Feels native.

Cons: Can push tabs down if many children. "×" buttons always visible might feel destructive.

C

Collapsible Section + Modal Management

Child grid: Collapsible "Sub-Projects" section above tabs. Starts expanded but can be collapsed to save space.

Management: Gear icon in section header opens a management modal. "Set Parent" available from project edit form.

Pros: User controls visibility. Scales to many children without pushing tabs off-screen. Clean.

Cons: Collapsed state might make children forgettable. Extra click to manage.

Recommendation: Approach C

Collapsible section scales best — ecosystems could have 2 or 20 children. The collapse toggle prevents layout thrashing while keeping children discoverable. The gear icon for management keeps the UI clean without hiding the capability.
